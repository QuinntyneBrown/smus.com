Boris Smus Projects About Archives

CrowdForge

Mechanical Turk is great for accomplishing small, well defined nuggets of work, (eg. labeling images), but terrible for many more complex and labor intensive real world tasks. Invented CrowdForge, a MapReduce-inspired framework for breaking down complex tasks into smaller sub-tasks. Presented at UIST 2011.

HTML5Rocks Articles

Writing articles about multi-touch web development, the Web Audio API, canvas performance, and high DPI images.

iWork

Engineering contributions to iWeb, iWork and iWork.com, a document sharing service for the web.

Mindstorms Projects

Built a few robots including an ink plotter, an Android-based crowd controlled vehicle, and a piano playing robot. Then co-organized a Mindstorms sumo wrestling competition at SXSW 2011.

Osmus

An open source sample multiplayer game written in JavaScript. The same game engine code runs on both client and server (Node.js). Heavily inspired by the excellent Osmos game for mobile devices.

Running Gestures

Facilitating hands-free gestures during physical activity, using an accelerometer inside the shoe. Presented at Ubicomp 2010.

Touch laptop experiments

An exploration into the touch laptop form factor. Responsive input switching between touch and mouse modes and simultaneous interactions with trackpad and touchscreen. I wrote some demos that illustrate these ideas.

Ubiquitous Drums

Drummers and non-drummers alike can often be seen making percussive gestures on their chests, knees and feet. Ubiquitous Drums enhances this experience by providing musical feedback for these and other gestures.

Web Audio book

A short book introducing the Web Audio API. Meant as an introduction to the web audio API, as well as some audio basics for web developers with little audio experience.

I'm Boris Smus, a software engineer and UI prototyper from Vancouver, Canada. I currently live in San Francisco and work on Chrome related things.
